# GitHub Pages 静态站说明

这是一个可直接部署到 **GitHub Pages** 的纯静态站点，当前包含：

- `index.html`：个人首页 / 资源入口页
- `reading.html`：阅读导航页
- `links.html`：后续 URL 链接预留页

## 1. 如何提交PR更新阅读页

阅读导航页是Markdown驱动，以后主要编辑`content/reading.md`即可。
`reading.html` 会自动加载并解析这个文件。

### Markdown 格式

建议保持下面这种结构：

```md
# 搜集到的参考阅读书目
简介：这里写页面开头说明
面向：研0
顺序：由易到难

## 通用主线
说明：这里写这个分组的简短说明

### How to Read a Paper
作者：S. Keshav
难度：★☆☆☆☆
状态：已整理
简介：这里写简短可见说明
我的想法：这里写你自己的补充判断
```

### 如果暂时还没读完

你可以这样写：

```md
### 某本书
作者：某作者
难度：★★★★☆
状态：未读完
简介：先写一个简短说明
我的想法：
```

这样页面会自动显示一个占位提示：

```text
还没读完，之后补。
```

### 如果还没开始读

可以写：

```md
状态：待读
我的想法：
```

页面会自动显示：

```text
还没开始读，之后补。
```

## 3. 本地预览

如果你本地有 Python，可以在当前目录运行：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

注意：因为 `reading.html` 要通过 `fetch` 读取 `content/reading.md`，所以**不要直接双击 HTML 文件打开**，而要用本地服务器预览。
